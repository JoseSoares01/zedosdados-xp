Sobre Mim

Olá! Sou o Zé dos Dados — designer e desenvolvedor apaixonado pela estética nostálgica dos anos 2000. Criei o MitchIvin XP como uma vitrine que combina o charme visual do Windows XP com práticas modernas de design de interfaces.

Trabalho na interseção entre design gráfico, desenvolvimento front-end e experiência do usuário. Acredito que interfaces bem construídas contam histórias — e nada conta histórias melhor do que um sistema operacional que marcou uma geração inteira.

MitchIvin XP

MitchIvin XP incorpora a estética nostálgica do design de sistemas operacionais do início dos anos 2000, reimaginada sob a ótica de um designer visual moderno. O sistema celebra cores primárias fortes — azuis vívidos, laranjas e verdes limão — contrastando com tons neutros marcantes e brancos cristalinos.

DESIGN EM RESUMO

  • Tahoma para textos altamente legíveis; MS Sans Serif para botões do sistema
  • Espaçamento baseado em grade de 4px para alinhamento
  • Bordas arredondadas nítidas limitadas a 3px (elementos) e 5px (superfícies)
  • Cores vibrantes do tema e estilo de elevação sutil em tons de azul

Este sistema equilibra geometrias retrô pixel-perfect com sofisticação contemporânea, criando uma interface que parece divertidamente familiar e intencionalmente refinada.

Lista de Boas Práticas (Fazer e Não Fazer)
Fazer
  • Use Verde Limão (#96FF96) especificamente para indicadores de sucesso.
  • Alinhe o espaçamento exclusivamente à grade base de 4px.
  • Limite o tamanho do raio das bordas dos componentes a 3px (elementos) e 5px (superfícies).
  • Empregue Tahoma para as hierarquias visuais principais.
  • Mantenha altas taxas de contraste de cor tipográfica (acima de 4,5:1).
Não Fazer
  • Não misture fontes serifadas e não-serifadas dentro da mesma página de layout.
  • Não exceda a largura padrão de botões de desktop (94px) sem contexto direto.
  • Evite múltiplas camadas de sombra sobrepostas.
  • Não use sistemas de redimensionamento de texto fluidos ou relativos.
  • Nunca use cores de destaque como amarelo ou verde para fundos de containers amplos.
Simulador de Breakpoints Responsivos

Simule como os componentes de layout respondem aos limites de tamanho de tela. Abaixo de `640px` de largura, as colunas colapsam em linhas únicas, o espaçamento diminui e os botões ocupam 100% de largura.
